# MEETING ANALYSIS REPORT

## SUMMARY
This meeting discusses the development of a data browser/interface for a meeting corpus analysis system. The team addresses prototype requirements for week six, debates the architectural approach for handling data (XML vs. database), and explores how to implement multi-level summarization and display functionality. Key decisions emerge around prioritizing the NITE XML framework, adopting an information density approach for summarization, and establishing a unified data abstraction model. The discussion is technical and exploratory, with participants working through design trade-offs.

---

## KEY DECISIONS

- **Prototype Timeline**: Prototype required for week 6 using pre-annotated data and baseline components where needed
- **Framework Choice**: Adopt NITE XML framework for core functionality due to its event synchronization and display element features
- **Data Storage Approach**: Use XML representation (not SQL database) for the browser display to leverage NITE XML's built-in synchronization capabilities
- **Summarization Strategy**: Calculate information density off-line for all utterances, then use this metric to drive multi-level summaries rather than creating separate summaries at different levels
- **Results Format**: Store calculation results in NITE XML format with timestamps for easy integration
- **Display Logic**: Implement zoom-level approach where double-clicking moves between detail levels (meeting series → individual meeting → segments) without requiring recalculation
